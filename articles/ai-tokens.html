AI Tokens — How LLMs Process Text

Stanimir Dimitrov June 2026 ~8 min read

Large Language Models don't read words — they read tokens. Understanding what a token is, how tokenization works, and how it affects cost, context, and model behaviour is fundamental to working effectively with any AI system.

What is a token?

A token is not a word, not a character, and not a syllable. It's a chunk of text that a language model has learned to recognize as a meaningful unit during training — the atomic unit of text for an LLM.

Rule of thumb: 1 token ≈ 4 characters in English, or roughly ¾ of a word. 100 tokens ≈ 75 words.

The sentence "Hello, how are you?" breaks into 6 tokens:

Text pieceToken count
Hello1
,1
how (space included)1
are1
you1
?1

How tokenization works — BPE

Most modern LLMs use Byte Pair Encoding (BPE), originally invented for data compression. During training it builds a vocabulary by iteratively merging the most frequent adjacent character pairs:

  1. Start with every character as its own token: ['h','e','l','l','o']
  2. Count all adjacent pairs across the training corpus
  3. Merge the most frequent pair into a new token
  4. Repeat millions of times until vocabulary size is reached (32k–100k tokens)
Iteration 1: 'e','r'  appear 50,000× → merge → 'er'
Iteration 2: 'er','s' appear 40,000× → merge → 'ers'
Iteration 3: 't','h'  appear 35,000× → merge → 'th'
...continuing until vocabulary is full

Tokenization in practice

Common words = 1 token

"the"      → [the]       = 1 token
"running"  → [running]   = 1 token
"Python"   → [Python]    = 1 token

Rare or long words = multiple tokens

"cryptocurrency" → [crypto][currency]    = 2 tokens
"tokenization"   → [token][ization]      = 2 tokens
"unbelievable"   → [un][believ][able]    = 3 tokens

Numbers are expensive

"2024"     → [2024]          = 1 token  (memorised as common year)
"19847362" → [198][47][362] = 3 tokens
"3.14159"  → [3][.][14][159]= 4 tokens

Whitespace is part of the token

"hello"  → [hello]   ← token ID 15339
" hello" → [ hello]  ← token ID 23748  (completely different!)

The leading space is encoded into the token itself. This is why prompt formatting matters.

Languages are NOT equal in token cost

English is the most efficient because BPE was trained primarily on English text. For the same meaning, other languages require significantly more tokens:

LanguagePhraseApprox. tokens
English"I love artificial intelligence"~5
French"J'aime l'intelligence artificielle"~8
Bulgarian"Обичам изкуствения интелект"~12
Arabic"أحب الذكاء الاصطناعي"~10
Chinese"我爱人工智能"~7

Cyrillic characters are split into more sub-pieces because they appear less in training data. Prompting in Bulgarian costs roughly 2–3× more tokens than the equivalent English prompt.

Context windows — your token budget

Every model has a context window — the maximum total tokens (input + output) it can process at once. Exceeding it causes truncation or errors.

How tokens affect API cost

APIs charge per token. Output tokens (the model's generated response) are always more expensive than input tokens because they are generated sequentially.

# Example — Claude Sonnet pricing
Input tokens:  $3.00 per 1M tokens
Output tokens: $15.00 per 1M tokens

Your message: "Explain tokenization in detail" = ~6 tokens
Model response: ~2,000 tokens

Cost = (6 × $0.000003) + (2,000 × $0.000015)
     = $0.000018 + $0.030
     ≈ $0.03 per call

How generation works — one token at a time

The model doesn't generate a full sentence at once. It predicts one token at a time, each time using all previous tokens as context:

Input:  "The capital of France is"
Step 1: predict → " Paris"   (probability: 94%)
Step 2: predict → "."        (probability: 78%)
Step 3: predict → " It"      (probability: 45%)
...

At each step, the model outputs a probability distribution over its entire vocabulary (~100,000 options). Which token is chosen depends on the sampling strategy:

StrategyHow it worksEffect
Greedy (temp=0)Always pick highest probabilityDeterministic, repetitive
Temperature > 1Flatten probabilitiesMore random, creative
Temperature < 1Sharpen probabilitiesFocused, conservative
Top-p (nucleus)Sample from tokens totalling p%Most used in production

Temperature = 0 → always identical output (useful for deterministic test assertions). Temperature = 1 → natural variation. Most production systems use 0.2–0.8.

Tokenization quirks that affect QA

Why LLMs struggle with arithmetic

The number 9.11 is tokenized as [9][.][11] — a sequence of text characters, not a decimal value. The model has no numeric understanding, which is why LLMs systematically fail at precise arithmetic without a calculator tool.

Capitalization creates different tokens

"Python"  → token ID 37474 = 1 token
"python"  → token ID 29356 = 1 token  (completely different ID!)
"PYTHON"  → [PY][THON]    = 2 tokens

Code is more efficient than prose

def add(a, b):          # ~10 tokens
    return a + b

"Write a Python function that takes two numbers as
parameters and returns their sum"  # ~22 tokens

This is why including code examples in prompts is often more token-efficient than explaining in prose.

Tools to count tokens

print(f"Count: {len(tokens)}")   # → 6
print(f"IDs:   {tokens}")        # → [15339, 11, 1268, 527, 499, 30]

Online tools: platform.openai.com/tokenizer and tiktokenizer.vercel.app visualize tokenization with color-coding per token.

Key takeaways for QA & AI engineers

  • Token count = cost + context limit — always estimate before building AI features
  • Non-English content costs 2–3× more tokens — plan your budget accordingly
  • Output tokens are 3–5× more expensive than input tokens
  • Temperature 0 gives reproducible responses — important for consistent test assertions against AI outputs
  • LLMs cannot reliably count, sort, or do arithmetic — these are inherent tokenization limitations
  • Large codebases or long conversation histories consume context quickly — chunk them appropriately
